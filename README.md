### 2. 브랜치 관리
- 각 과제는 별도 브랜치에서 작업하고 순차적으로 main에 병합합니다.

```bash
git checkout -b feature/test1
# 작업 및 커밋
git checkout main
git merge feature/test1 --ff
git checkout -b feature/test2
# 다음 작업 진행...
```

- git 커밋 수 제한 없음 : 여러 단계로 작업을 하셨다면 제출하실 때 git 중간 커밋들을 squash 하시기 보단 그대로 남겨두시길 권장합니다.
